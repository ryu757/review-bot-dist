# Google レビュー自動返信システム

Google マップのレビューを自動取得し、AI が返信を下書き → スプレッドシートで確認 → 「投稿する」に変えるだけで Google マップに自動投稿されます。

返信は**ホームページの内容を参照して生成**されるため、書かれていない情報を捏造しません。

---

| ファイル | 用途 |

| **`① 最初にお読みください.md`** | このご案内 |
| **`② Mac セットアップ.command`** / `② Windows セットアップ.bat` | お使いの OS の方を**ダブルクリック**してセットアップ |
| `Mac 自動実行の停止.command` 等 | 自動実行を止めたい時用（普段は使わない） |
| `システム（触らないでください）` | プログラム本体（**そのまま置いてください**） |

## ✅ 事前にご用意いただくもの

- **Google アカウント**（ビジネスプロフィールの管理者権限を持つもの）
- **クレジットカード**（Anthropic API 用・月数十〜数百円）
- 自社ホームページの URL（あれば。AI が参照します）
- パソコン（Mac / Windows / Linux）

# 🛠 セットアップ（合計 30〜40 分）

セットアップは大きく **3 つの段階** に分かれます：

- **段階 A**: Anthropic API キー取得（5 分）
- **段階 B**: Google Cloud で OAuth クライアント作成（15〜20 分）
- **段階 C**: ウィザード実行（10 分）

---

## 段階 A：Anthropic API キーを取得（5 分）

返信文を AI に書かせるための API キーです。

### A-1. アカウント作成

1. <https://console.anthropic.com> をブラウザで開く
2. 「Sign up」または Google ログインでアカウント作成

### A-2. クレジット入金（必須）

API キーを使うには **クレジットを先にチャージ**する必要があります。

1. 左メニューの **「Billing」**（課金）をクリック
2. **「Add payment method」** でクレジットカードを登録
3. **「Add Credits」** で最低 $5（約 750 円）入金
   - 月数十円〜数百円の使用量なので、$5 で何ヶ月も持ちます

### A-3. API キーを発行

1. 左メニューの **「API Keys」** をクリック
2. 右上の **「Create Key」** をクリック
3. キー名は何でも OK（例: `review-bot`）→ 「Create Key」
4. **`sk-ant-api03-...` で始まる長い文字列** をコピー
   - ⚠️ **このキーは一度しか表示されません**。必ずコピーして安全な場所に保管
5. 後で段階 C のスプレッドシートに貼り付けます

---

## 段階 B：Google Cloud で OAuth クライアント作成（15〜20 分）

スプレッドシートにアクセスする許可をシステムに与えるための準備です。

### ⚠️ 個人 Google アカウント（@gmail.com）を推奨

会社の Google Workspace アカウントだと **「請求先アカウント必須」「親リソース必須」** といったポリシーで止まることがあります。可能なら個人 Gmail アカウントで進めるのがスムーズです。

### B-1. プロジェクトを作る

1. <https://console.cloud.google.com/projectcreate> を開く
2. **プロジェクト名**: 何でも OK（例: `review-bot`）
3. もし「組織」欄が表示されていたら：
   - ドロップダウンを開いて **「組織なし」** を選択
   - これで「請求先アカウント」「親リソース」の必須マークが消えます
4. **「作成」** をクリック → 30 秒ほど待つ

> 📷 **[スクリーンショット枠]**

### B-2. Google Sheets API を有効化

1. 画面上部の **検索バー** に **「Google Sheets API」** と入力
2. 検索結果から **「Google Sheets API」** をクリック
3. 青い **「有効にする」** ボタンをクリック → 数秒待つ

> 📷 **[スクリーンショット枠]**

### B-3. OAuth 同意画面の設定（新 UI: Google Auth Platform）

> 注：Google Cloud の UI は頻繁に変わります。表示が違っても、似た意味のボタンを探してください。

1. 左メニュー **「APIとサービス」** → **「OAuth 同意画面」**
2. 「Google Auth Platform はまだ構成されていません」が出たら、青い **「開始」** をクリック
3. 順に入力：
   - **アプリ名**: 何でも OK（例: `Review Reply Bot`）
   - **ユーザーサポートメール**: 自分のメールアドレス
   - **オーディエンス（対象）**: **「外部」** を選択
   - **デベロッパーの連絡先**: 自分のメールアドレス
4. **「Google API サービス利用規約に同意する」** にチェック → **「続行」** または **「作成」**

### B-4. テストユーザーを追加（重要）

OAuth 同意画面の設定後、左メニュー **「対象」**（または「テストユーザー」）をクリック

1. **「+ Add users」** または **「+ ユーザーを追加」** をクリック
2. **自分のメールアドレス** を入力 → **「追加」** → **「保存」**

> ⚠️ ここで自分のメールを追加しないと、後で OAuth 認証時に「未承認のアプリ」エラーで進めません。

### B-5. OAuth クライアント ID を作成

1. 左メニュー **「クライアント」**（または「APIとサービス」→「認証情報」）
2. **「+ クライアントを作成」** または **「+ 認証情報を作成」→「OAuth クライアント ID」**
3. **アプリケーションの種類**: **「デスクトップアプリ」** を選択
4. **名前**: 何でも OK（例: `desktop`）→ **「作成」**

### B-6. JSON をダウンロード

1. 作成完了後、画面右側または下部の **「JSON をダウンロード」** をクリック
2. ダウンロードフォルダに `client_secret_xxxxx.json` が保存されます
3. **ファイル名や場所は変えなくて OK**（ウィザードが自動で見つけます）

## 段階 C：ウィザード実行（10 分）

### C-1. セットアップ起動

お使いの OS のファイルを**ダブルクリック**：

- **Mac**: `② Mac セットアップ.command`
- **Windows**: `② Windows セットアップ.bat`
- **Linux**: `② Linux セットアップ.sh`

#### Mac で「開発元未確認」が出た場合

新しい macOS では **右クリック → 開く** だけでは進めない場合があります。その時は：

1. 警告ダイアログの「OK」を押して閉じる
2. **アップルメニュー → システム設定 → プライバシーとセキュリティ**
3. 下にスクロール → **「"② Mac セットアップ.command" は使用がブロックされました」** の横の **「このまま開く」**
4. パスワード or Touch ID で承認 → 確認ダイアログで **「開く」**

### C-2. ターミナルが開いて自動進行

ウィザードが Python 環境構築・依存ライブラリインストール・OAuth クライアント JSON 自動検出 を順に実行します。途中で何度か質問が出るので **Y** を押すか ENTER で進めてください。

### C-3. スプレッドシートに 4 項目を入力

ウィザードが自動でブラウザを開きます。「**設定**」タブで以下 4 項目を入力 → ターミナルに戻って ENTER。

| 項目 | 例 |
|---|---|
| 企業名 | 鈴木自動車整備工場 |
| 業界 | 自動車整備 |
| ホームページURL | https://example.com |
| Anthropic APIキー | （段階 A でコピーした `sk-ant-...`） |

### C-4. Google ビジネスプロフィールにログイン

ブラウザが自動で開きます。**返信を投稿する Google アカウント**でログインしてください。

> ⚠️ ここでログインするアカウントは、**Google ビジネスプロフィールの管理者権限**を持つアカウントです。段階 B の OAuth で使ったアカウントと同じでも別でも構いません（権限さえあれば）。

「未確認のアプリ」と出た場合は **「詳細」→「（アプリ名）に移動」**（テストユーザーに登録済みなら正常）。

ログイン完了後、ブラウザを閉じれば OK。

### C-5. 初回取得の動作確認

ウィザード最終ステップで「今すぐ実行しますか？」→ **Y** を押すと、すぐに最初のレビュー取得が走ります。スプレッドシートの「レビュー返信」タブにレビューと AI 下書きが入ります。

「セットアップ完了！」と表示されたら終了です。

## 🔄 日常の使い方

セットアップ後は**何もしなくても自動で動きます**。

1. 30 分ごとに新着レビューを自動取得 → スプレッドシートに下書き追加
2. スプレッドシートを確認 → 必要なら下書きを編集
3. 投稿する行の「**ステータス**」列のセルを **クリック → プルダウンから「投稿する」を選択**
4. 10 分以内に自動投稿 → ステータスが「投稿済み」に変わる

### スプレッドシートの 2 つのタブ

- **「レビュー返信」タブ**: 下書きとステータスの確認・編集
- **「設定」タブ**: 企業名・業界・URL・APIキーの変更（次回 sync から反映）

ホームページの内容は **24 時間ごとに自動再取得** されます。

## 🆘 困ったとき

| 状況 | 対処 |
|---|---|
| Cloud Console で「請求先アカウント必須」エラー | 組織ドロップダウンを **「組織なし」** に変更（または個人 Gmail で作り直し） |
| Google ログイン時「未確認のアプリ」 | **「詳細」→「（アプリ名）に移動」** をクリック（テストユーザー登録済みなら正常） |
| Mac で .command が「開けません」 | システム設定 → プライバシーとセキュリティ → **「このまま開く」**（C-1 参照） |
| Anthropic API キーが「Insufficient credits」 | <https://console.anthropic.com/settings/billing> でクレジット入金 |
| 新着が取れない / 投稿失敗 | `Mac 自動実行の停止.command` → `② Mac セットアップ.command` を再実行（ログイン更新） |
| 自動実行を止めたい | `Mac 自動実行の停止.command`（または Windows 版）をダブルクリック |
| 企業情報を変えたい | スプレッドシート「設定」タブを直接編集 → 次回 sync から反映 |
| ホームページ情報をすぐ更新したい | `システム（触らないでください）/credentials/website_cache.json` を削除 |

---

## ⚠️ 注意

- 自動実行は PC が起動している間だけ動きます（スリープ中は止まり、起動時に自動で catch-up）
- Anthropic API は使用量に応じ課金（レビュー1件あたり約 0.5 円）
- Google ログインセッションは数週間〜数ヶ月で切れます（切れたら再セットアップで再ログイン）
